HOW IT WORKS:
───────────────────────────────────────────────────────────────

1. FRONTEND LOADS
   └─ frontend/index.html opens in browser
   └─ frontend/js/user-auth.js initializes
   └─ Checks for existing JWT token in localStorage

2. USER LOGS IN
   └─ Frontend POSTs email/password to /api/auth/login
   └─ Backend verifies credentials against SQLite
   └─ Returns JWT token + user data
   └─ Frontend stores token in localStorage
   └─ Subsequent requests include "Authorization: Bearer {token}"

3. BACKEND VALIDATES
   └─ middleware/auth.js middleware checks token validity
   └─ Verifies token signature with JWT_SECRET
   └─ Checks session hasn't been revoked
   └─ Confirms user is active
   └─ Attaches user data to request object

4. APP FUNCTIONALITY
   └─ Dashboard loads with user data
   └─ Portfolio data pulled from /api/wallet
   └─ Trading history from /api/trades
   └─ All requests authenticated via JWT

───────────────────────────────────────────────────────────────
AVAILABLE API ENDPOINTS:
───────────────────────────────────────────────────────────────

Authentication:
  POST   /api/auth/register          - Create account
  POST   /api/auth/login             - Login
  GET    /api/auth/me                - Current user + wallet
  POST   /api/auth/logout            - Revoke session
  POST   /api/auth/change-password   - Change password

Wallet:
  GET    /api/wallet                 - Get wallet balance
  POST   /api/wallet/deposit         - Add funds
  POST   /api/wallet/withdraw        - Withdraw funds
  GET    /api/wallet/transactions    - Transaction history

Trades:
  POST   /api/trades                 - Log trade
  GET    /api/trades                 - List user's trades
  GET    /api/trades/stats           - Win rate, PnL stats
  GET    /api/trades/open            - Open positions
  PATCH  /api/trades/:id/close       - Close a trade

Admin Panel:
  GET    /api/admin/users            - All users (admin only)
  GET    /api/admin/transactions     - All transactions
  PATCH  /api/admin/users/:id        - Update user

Payments (Stripe):
  GET    /api/stripe/publishable-key - Get Stripe key
  POST   /api/stripe/create-session  - Create checkout

KYC Verification:
  POST   /api/kyc/submit             - Submit documents
  GET    /api/kyc/status             - Check status

───────────────────────────────────────────────────────────────
TROUBLESHOOTING:
───────────────────────────────────────────────────────────────

Problem: "Server unreachable" error
Solution: Ensure backend is running with: cd backend && npm start

Problem: CORS errors
Solution: Check ALLOWED_ORIGINS in server.js includes your frontend URL

Problem: JWT token invalid
Solution: Clear browser localStorage and login again

Problem: Password too weak
Solution: Backend enforces: 12+ chars, uppercase, lowercase, number, special char
         Use: ZenAdmin2026! or similar

Problem: Database locked
Solution: Restart server - SQLite WAL file might be corrupted
